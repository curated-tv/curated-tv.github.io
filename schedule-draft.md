## Hard Curation Rules

These are now treated as non-negotiable scheduling rules.

1. **18:00-21:00 music / art / drama slot**
   - Allowed: Bengali classic song traditions performed/recorded before the 1960s, especially Rabindra Sangeet, Nazrul Geeti, Atulprasadi, Rajanikanta, Dwijendrageeti, and closely related early Bengali song forms.
   - Allowed: North Indian / Hindustani classical music.
   - Preferred evidence: title/uploader metadata says 78 rpm, shellac, 1930s, 1940s, 1950s, early radio, or otherwise clearly archival.
   - Not allowed: modern pop, modern covers, generic world music, NPR Tiny Desk, Western classical, generic art explainers, or theatre snippets that are not part of the Bengali/classical rule-set.
2. **Movie / drama slot**
   - Allowed: old Bengali films and Bengali drama-style films from before 1960.
   - Exception: Satyajit Ray films may be used even after 1960.
   - Not allowed in the core schedule: non-Bengali films, even if classic. This excludes Chaplin's *Modern Times* from the movie slot.
3. **Suggestions**
   - New videos can be added freely, but only if they do not violate these rules.
   - Ambiguous uploads should be marked `verify before final month schedule`.

Open clarification:

- The original daily wheel has a separate `01:00-05:00` lo-fi slot. This draft keeps that slot because it was explicitly part of the original schedule.
- If the new music rule is meant to override **all** music, including overnight lo-fi, replace the lo-fi slot with night ragas, dhrupad, alap, or other quiet Hindustani classical programming.

## Next Implementation Step

Replace the flat `videos` loop with a clock-slot schedule:

```js
schedule: [
  {
    start: "18:00",
    end: "21:00",
    label: "Bengali classics / Hindustani classical",
    videos: [
      { id: "toJnqAL_nZs", durationSeconds: 3284 },
      { id: "bWr_hoSEczw", durationSeconds: 183 },
      { id: "EXeXnKiKg78", durationSeconds: 2742 },
    ],
  },
]
```

The player should:

1. Find the current IST slot.
2. Calculate elapsed seconds inside that slot.
3. Loop only through that slot's videos.
4. Retune when a slot changes.
5. Support a future `monthSchedule`, where each calendar day points to a different set of blocks.
